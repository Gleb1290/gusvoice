# 🪿 GusVoice

[English](README.md) · **Русский**

**Свой сервер для голоса, видео, демонстрации экрана и текстового чата — для тебя и твоих друзей.**
Как Discord / TeamSpeak, только крутится на *твоём* сервере, поднимается одним `docker compose`, а все секреты генерируются за тебя.

GusVoice даёт компании друзей своё личное место для общения: **серверы → категории → текстовые и голосовые каналы**, роли и права в стиле Discord, личные сообщения, реакции, кастомные звуки, встроенное шумоподавление, двухфакторка и **фоновые пуши** на Android **без Google/FCM**. Весь звук и видео проходит через собственный **[LiveKit](https://livekit.io) SFU** — никакого peer-to-peer, так что ничей IP не светится остальным в канале.

Весь стек — `postgres · redis · livekit · backend · presence · веб-клиент`, плюс встроенные **Caddy** (авто-HTTPS), **MinIO** (медиа) и **ntfy** (пуши) — поднимается из одного compose-файла.

## Что умеет

- 🎙️ **Голос, видео и демонстрация экрана** с настройкой качества/FPS по каналам и живым списком тех, кто сейчас говорит.
- 💬 **Текстовый чат** — реакции, ответы, @упоминания, бейджи непрочитанного/упоминаний, вложения файлов, поиск по сообщениям.
- 🛡️ **Роли и права** с переопределениями по каналам и категориям, серверная модерация голоса и журнал аудита.
- 📱 **Одно приложение — твой сервер** — веб, самообновляющийся десктоп **Windows** и **Android**. Адрес сервера вводишь при входе; сменить сервер можно в любой момент.
- 🔔 **Фоновые уведомления** о ЛС/@упоминаниях на Android через UnifiedPush — без Google Play Services.
- 🔒 **Безопасно по умолчанию** — каждый секрет генерируется при установке, fail-closed конфиг, rate-limit на авторизацию, опциональная 2FA, анти-бот защита регистрации.
- 📦 **Всё в комплекте** — встроенные TLS, объектное хранилище и пуш-шлюз. Ни на что внешнее регистрироваться не нужно.

## Что нужно

- **Linux-хост** (небольшой VPS или домашняя машина) с **~2 ГБ свободной RAM**.
- **Домен**, которым ты управляешь (GusVoice использует шесть поддоменов: `voice` · `api` · `presence` · `lk` · `media` · `ntfy`).
- Возможность **открыть несколько портов** (`80`, `443` и медиа LiveKit `7881/tcp` + `7882/udp`).
- **Docker** — установщик предложит поставить его сам, если его нет.

## Установка

```bash
# одной командой — скачает всё, задаст пару вопросов и поднимет стек
curl -fsSL https://raw.githubusercontent.com/Gleb1290/gusvoice/main/install.sh | bash
```

или, если хочешь сначала склонировать и прочитать перед запуском:

```bash
git clone https://github.com/Gleb1290/gusvoice.git
cd gusvoice
./install.sh
```

Установщик **генерирует все секреты**, пишет `.env` и поднимает стек за встроенным Caddy с автоматическим HTTPS от Let's Encrypt (или остаётся за уже работающим у тебя обратным прокси — он спросит). В конце он печатает точные **DNS-записи** и **проброс портов**, которые нужно настроить. Полное руководство — включая подключение десктоп- и Android-клиентов — в **[docs/SELF-HOSTING.md](docs/SELF-HOSTING.md)**.

Когда всё готово, открывай `https://voice.<твой-домен>` и заходи под супер-админом, которого он создал.

## Обновление

Забирает свежие серверные образы и файлы стека — одной командой откуда угодно:

```bash
curl -fsSL https://raw.githubusercontent.com/Gleb1290/gusvoice/main/update.sh | bash
```

или из папки установки:

```bash
cd gusvoice
./update.sh
```

`update.sh` делает бэкап текущих `docker-compose.yml` + `config/`, обновляет их до последнего релиза, тянет новые образы и пересоздаёт только изменившиеся контейнеры. **Твой `.env` и все данные остаются нетронутыми**, а бэкенд сам применяет ожидающие миграции БД при старте.

| Флаг | Что делает |
|------|------------|
| `--images-only` | Только тянет образы + пересоздаёт — не трогает `docker-compose.yml` / `config/`. |
| `--prune` | После обновления удаляет старые висящие образы, чтобы освободить диск. |
| `-y`, `--yes` | Запуск без вопросов. |

> Десктоп и Android **обновляются сами** — тебе нужно обновлять только сервер.

## Удаление

```bash
cd gusvoice
./uninstall.sh            # остановить и удалить контейнеры — ДАННЫЕ сохраняются
```

Это останавливает GusVoice, но сохраняет базу, загрузки и секреты, так что `./install.sh` (или `./update.sh`) позже продолжит ровно с того места, где ты остановился.

Чтобы стереть **всё** — все аккаунты, сообщения, загрузки и секреты — навсегда:

```bash
./uninstall.sh --purge    # попросит ввести «yes»; затем удалит тома с данными + .env
```

## Клиенты

Скачай десктоп- или Android-приложение со **[страницы релизов](https://github.com/Gleb1290/gusvoice/releases/latest)**, запусти и введи адрес своего сервера (например, `voice.example.com`). Сменить сервер можно в любой момент в **Настройки → Аккаунт → Сервер**. Веб-клиент доступен на `https://voice.<твой-домен>` — качать ничего не нужно.

## Где что лежит

| Что | Где |
|-----|-----|
| Твой конфиг и **все секреты** | `.env` (генерируется установщиком, `chmod 600` — никогда не коммить) |
| База, медиа, пуши и состояние TLS | Docker **именованные тома** (`gusvoice_pgdata`, `gusvoice_miniodata`, `gusvoice_ntfydata`, `gusvoice_caddydata`, …) |
| Шаблоны обратного прокси / TLS | `config/Caddyfile`, `config/livekit.yaml` (управляются через `.env`) |
| Бэкапы обновлений | `./.gusvoice-backups/<метка-времени>/` |

Чтобы настроить с нуля, удали `.env` и перезапусти `./install.sh`. Чтобы работать за **своим** обратным прокси (Nginx Proxy Manager / nginx / Traefik) вместо встроенного Caddy — установщик спросит; подробности в [docs/SELF-HOSTING.md](docs/SELF-HOSTING.md).
